# FinanceSystem 개선 사항 및 변경 내역

## 1. 🐛 핵심 버그 수정 및 DB 변경
* **`schema.sql`**: `transaction` 테이블명을 MySQL 예약어 충돌 방지를 위해 `transaction_history`로 변경
* **`TransactionMapper.xml`**: 테이블명을 `transaction_history`로 맞추고 정렬 기준을 `ORDER BY created_at DESC`로 변경
* **`TransactionServiceImpl.java`**: DB 저장 시 거래 시간 누락(Null) 방지를 위해 `LocalDateTime.now()` 주입 로직 추가
* **`TransactionController.java`**: 입출금 완료 시 존재하지 않는 `Result` 페이지 대신 `redirect:/transactions`로 이동하도록 수정
* **`login.html`, `signup.html`**: 백엔드 연동을 위해 폼 태그의 ID 입력 변수를 `name="userid"`로 변경

## 2. 💄 UI/UX 및 기능 개선
* **`TransactionController.java`, `MemberController.java`**: HTML단과 매칭되도록 에러 변수명을 `errorMessage`에서 `error`로 변경
* **`MemberService.java`, `MemberServiceImpl.java`**: 회원가입 함수 리턴 타입을 `String`으로 변경하여 랜덤 생성된 계좌번호 반환
* **`MemberController.java`**: 회원가입 성공 후 리다이렉트 시 `RedirectAttributes`로 계좌번호를 전달하여 화면에 알림창 출력
* **`header.html`, `styles.css`**: 기존 우측 상단 토스트 알림을 화면을 덮는 **중앙 모달(Modal)창**으로 개편 (어두운 배경, 흑백 톤, 가운데 버튼)

## 3. ⚙️ 비즈니스 로직 및 인프라(Docker) 설정
* **`TransactionServiceImpl.java`**: 출금 금액이 1,000,000원을 초과할 경우 에러를 발생시키는 1회 한도 제한 로직 추가
* **`Dockerfile`**: 단순 문법 에러 수정
* **`docker-compose.yml`**: MySQL 및 백엔드 컨테이너 구성 작성. 거래 시간 오류 수정을 위해 컨테이너에 `TZ: Asia/Seoul` 환경 변수 추가
* docker-compose up -d 로 실행 가능.
